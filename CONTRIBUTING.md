# Contributing to IntentFlow

Thank you for your interest in contributing to IntentFlow!

## Getting Started

### Fork the Repository

1. Fork the repository on GitHub.
2. Clone your fork:

```bash
git clone https://github.com/<your-username>/intentflow.git
cd intentflow
```

3. Create a branch:

```bash
git checkout -b feature/my-change
```

## Development Principles

IntentFlow is designed to remain:

* Lightweight
* Dependency-free whenever possible
* Easy to understand
* Backward compatible

Please avoid introducing unnecessary complexity.

## Code Style

* Keep functions small and focused.
* Use meaningful variable names.
* Prefer readability over clever implementations.
* Add comments only when they improve understanding.

## Testing

Before submitting a pull request:

* Run all existing tests.
* Ensure new changes do not break current functionality.
* Add tests for new features when applicable.

## Pull Requests

When opening a PR:

1. Describe the problem being solved.
2. Explain your approach.
3. Reference any related issues.
4. Keep PRs focused on a single change.

## Reporting Issues

When reporting bugs, include:

* Expected behavior
* Actual behavior
* Steps to reproduce
* Environment information

## Community Expectations

Be respectful and constructive when interacting with other contributors.

Thank you for helping improve IntentFlow!
